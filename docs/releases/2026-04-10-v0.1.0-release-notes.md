# Release Notes - v0.1.0

## Summary

`v0.1.0` establishes the first coherent product-management backend release for
the project.

It includes:

- admin product management
- storefront product search
- inventory reservation, confirmation, adjustment, and ledger history
- price history and scheduled pricing
- header-based auth and merchant scope control
- Docker packaging and backend CI

## Highlights

### Product and Search

- Added a storefront search projection backed by MySQL.
- Reduced storefront search coupling to transactional tables.
- Preserved the existing `GET /products` API shape while changing the read path.

### Inventory and Pricing

- Added immutable inventory ledger history.
- Added automatic due-price execution through the in-process scheduler.
- Kept manual admin controls for inventory and pricing operations.

### Security and Delivery

- Added request-header based admin authentication and merchant isolation.
- Added Swagger/OpenAPI UI and backend test CI.
- Added container packaging assets for local delivery and demo setups.

## Verification

Verified with:

```powershell
cd backend
.\gradlew.bat clean test --no-daemon
```

## Known Limitations

- Storefront visibility still needs a full publish/audit workflow.
- Authentication still uses a lightweight header model.
- Scheduled pricing assumes a single running application instance.
